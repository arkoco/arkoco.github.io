code-serve

下面给你一套在 Debian 12（x64）云服务器上，直接安装 code-server，并完整配置 Git / JDK / Node.js / 版本管理工具，同时保证 code-server 能正确识别环境变量的可直接执行方案。 这套方案偏 长期使用 + 多版本 + 稳定，非常适合你这种全栈/Java 后端开发者。

一、基础系统准备（必做）​

sudo apt update
sudo apt upgrade -y

# 常用工具
sudo apt install -y curl wget git unzip vim ca-certificates gnupg lsb-release build-essential

二、安装 code-server（官方推荐方式）​

1️⃣ 添加 code-server 官方仓库​

curl -fsSL https://code-server.dev/install.sh | sh

安装完成后会自动生成 systemd 服务。

2️⃣ 启动并设置开机自启​

systemctl --user enable code-server
systemctl --user start code-server

⚠️ 注意：code-server 默认运行在 ~/.config/code-server/

3️⃣ 查看访问密码​

cat ~/.config/code-server/config.yaml

默认监听：

http://127.0.0.1:8080

如果你后面用 Cloudflare Tunnel / Nginx / SSH 隧道，这里不用改。

三、安装 Git（最新版，Debian 官方）​

sudo apt install -y git
git --version

基础配置（建议）：

git config --global user.name "your-name"
git config --global user.email "your-email@example.com"
git config --global core.editor "code --wait"

四、安装 JDK（使用 SDKMAN，强烈推荐）​

原因：

  • 支持多版本 JDK（8 / 11 / 17 / 21）
  • 对 code-server / IDEA CLI / Maven 友好
  • 不污染系统环境

1️⃣ 安装 SDKMAN​

curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"

验证：

sdk version

2️⃣ 安装常用 JDK​

# Java 8
sdk install java 8.0.402-tem

# Java 11
sdk install java 11.0.22-tem

# Java 17（推荐）
sdk install java 17.0.10-tem

# Java 21（新项目）
sdk install java 21.0.2-tem

切换版本：

sdk use java 17.0.10-tem

3️⃣ Maven / Gradle（可选）​

sdk install maven
sdk install gradle

五、安装 Node.js（使用 nvm）​

1️⃣ 安装 nvm​

curl -fsSL https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc

验证：

nvm --version

2️⃣ 安装 Node.js 多版本​

# LTS
nvm install --lts
nvm use --lts

# 指定版本
nvm install 18
nvm install 20

设置默认版本：

nvm alias default lts/*

六、确保 code-server 识别所有环境变量（关键）​

1️⃣ code-server 默认 不加载 ~/.bashrc​

必须手动指定 shell 环境。

编辑配置：

vim ~/.config/code-server/config.yaml

推荐配置：

bind-addr: 127.0.0.1:8080
auth: password
password: your-password
cert: false

七、在 code-server 里验证环境​

打开 code-server → Terminal：

java -version
mvn -version
node -v
npm -v
git --version

都能正常识别即成功 ✅

八、推荐额外安装（开发体验拉满）​

# zsh + oh-my-zsh（可选）
sudo apt install -y zsh
chsh -s $(which zsh)

# 常用 CLI
sudo apt install -y htop tmux ripgrep fd-find

七、正确做法二（强烈推荐）：项目级环境绑定（最稳）

这是 “我在任何机器 clone 都能跑” 的方式

Java 项目（Spring Boot 示例） .vscode/launch.json


{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "java",
      "name": "Spring Boot",
      "request": "launch",
      "mainClass": "com.example.Application",
      "env": {
        "JAVA_HOME": "/home/youruser/.sdkman/candidates/java/current"
      }
    }
  ]
}

Node 项目

{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Run Dev",
      "program": "${workspaceFolder}/index.js",
      "env": {
        "NODE_ENV": "development"
      }
    }
  ]
}

八、验证方式（一定要做）

在 code-server 里建一个任务：

.vscode/tasks.json

{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "env-check",
      "type": "shell",
      "command": "echo $JAVA_HOME && java -version && node -v"
    }
  ]
}

运行 Task：

Terminal → Run Task → env-check

如果成功 👉 你已经彻底解决环境问题

九、总结一句人话版

❌ 能在 Terminal 用 ≠ 项目能跑 ✅ code-server 主进程环境变量 才是关键 ✅ systemd environment + 项目级 env = 永久稳定方案
